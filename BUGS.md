# Pawen Command Center — Bug Log

Format: one H3 per bug. Never delete entries — flip `status` instead.

Severity levels: `CRITICAL` (data loss / security breach / payment bypass) · `HIGH` (feature broken for all users) · `MEDIUM` (feature broken for some / degraded UX) · `LOW` (cosmetic / edge case)

Status values: `open` · `in-progress` · `fixed` · `wontfix` · `duplicate`

---

## Template (copy below for each new bug)

### BUG-XXX — <one-line title>
- **Severity**: CRITICAL | HIGH | MEDIUM | LOW
- **File(s)**: `path/to/file.ts:LINE`
- **Discovered**: YYYY-MM-DD · iteration N
- **Status**: open
- **Repro**:
  1. step one
  2. step two
  3. step three
- **Expected**: what should happen
- **Observed**: what actually happens
- **Root cause**: (fill after investigation)
- **Fix**: (describe change; link commit when applied)
